■使用技術
・バックエンド：Java / Spring Boot / MyBatis
・フロントエンド：HTML / CSS / JavaScript
・インフラ：AWS（EC2）
・データベース：MySQL（または PostgreSQL）

■工夫した点
・Spring Securityを用いたログイン認証機能の実装
・MyBatisによるSQLの明示的な管理
・バリデーショングループを用いた入力制御の切り分け
・勤怠の実績・予定データから各種労働時間を自動計算するロジックの実装
・AWS EC2上へのデプロイおよび動作確認

■担当
・要件定義、設計、開発、テスト、AWS環境構築まで一貫して担当

■システム名
勤怠管理システム

■目的
社員の勤怠情報および交通費・経費申請を一元管理し、
入力・集計・承認に関する業務の効率化を図ることを目的とする。

また、手作業による計算ミスや申請漏れを防止し、
データの正確性および業務の可視化を実現する。

■背景
従来、自社では勤怠管理をExcelで行っており、
入力ミスや集計の手間、承認状況の不透明さといった課題があり、
業務効率および管理精度の低下が懸念されていた。

その後、勤怠管理アプリの導入が検討され、
一時的に新しいシステムに触れる機会があった。
（※最終的には法的な要件により導入は見送られた）

しかし、そのシステムが直感的で使いやすい設計であったことから、
自身でも同様の勤怠管理システムを開発してみたいと考え、本システムの開発に至った。



■対象ユーザー（想定）
・一般社員
・管理者

■利用環境
・PC（ブラウザ）※社内ネットワークから利用を想定
・インフラ：AWS EC2上にアプリケーションをデプロイ

■機能一覧

・ログイン機能
　ユーザーIDとパスワードにより認証を行う

・ユーザー登録機能
　ユーザーID、パスワード、ユーザー名、誕生日、年齢、性別を入力しユーザー登録を行う

・ユーザー編集機能（管理者用）
　パスワード、ユーザー名の変更とユーザー情報の削除を行う

・勤怠入力機能
　予定出退勤時間、予定休憩時間、実績出退勤時間、実績休憩時間の入力を行う

・勤怠一覧表示機能
　月ごとの勤怠情報を一覧で表示する

・勤怠自動計算機能
　所定時間・実働時間・休憩時間・時間外労働・控除時間・休日出勤・出勤日数・欠勤日数・有給休暇・振替休暇・代休・慶弔休暇・病欠・産休・育児休暇・その他休暇を自動計算する

・カレンダー勤怠登録済み表示機能
　勤怠登録済みの日はカレンダーの日付欄に「登録済み」と表示される

・交通費申請機能
　日付、交通手段、出発地、到着地、経由地、金額、特記事項を入力し交通費申請を行う

・経費申請機能
　日付、用途、金額、特記事項を入力し経費申請を行う

・承認機能
　管理者が申請内容を確認し、承認または差戻を行う

・通知機能
　申請・承認・差戻時に、対象ユーザーおよび管理者へメール通知を行う

・権限管理機能
　ユーザー権限に応じて利用可能な機能を制御する
