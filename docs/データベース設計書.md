## ■ テーブル設計概要

本システムでは、ユーザー情報・勤怠情報・交通費・経費・申請情報を管理するため、以下のテーブルで構成する。

---

### ■ m_user（ユーザーマスタ）

ユーザーの基本情報を管理するテーブル。

| カラム名      | 型            | 説明         |
| --------- | ------------ | ---------- |
| user_id   | VARCHAR(50)  | ユーザーID（PK） |
| password  | VARCHAR(100) | パスワード      |
| user_name | VARCHAR(50)  | ユーザー名      |
| birthday  | DATE         | 生年月日       |
| gender    | INT          | 性別         |
| role      | VARCHAR(50)  | 権限（管理者/一般） |

---

### ■ t_kintai（勤怠テーブル）

ユーザーごとの日次勤怠情報を管理するテーブル。

| カラム名                     | 型           | 説明             |
| ------------------------ | ----------- | -------------- |
| user_id                  | VARCHAR(50) | ユーザーID（PK, FK） |
| work_date                | DATE        | 日付（PK）         |
| planned_work_start_time  | TIME        | 予定開始時間         |
| planned_work_end_time    | TIME        | 予定終了時間         |
| planned_break_start_time | TIME        | 予定休憩開始         |
| planned_break_end_time   | TIME        | 予定休憩終了         |
| actual_work_start_time   | TIME        | 実働開始時間         |
| actual_work_end_time     | TIME        | 実働終了時間         |
| actual_break_start_time  | TIME        | 実働休憩開始         |
| actual_break_end_time    | TIME        | 実働休憩終了         |
| scheduled_work_hours     | DECIMAL     | 所定労働時間         |
| actual_work_hours        | DECIMAL     | 実働時間           |
| overtime_hours           | DECIMAL     | 時間外労働時間        |
| deduction_time           | DECIMAL     | 控除時間           |
| kintai_status_id         | INT         | 勤怠区分（有給・代休など）  |
| kintai_comment           | TEXT        | 特記事項           |

---

### ■ t_koutsuhi（交通費テーブル）

交通費申請情報を管理するテーブル。

| カラム名        | 型           | 説明         |
| ----------- | ----------- | ---------- |
| koutsuhi_id | SERIAL      | ID（PK）     |
| user_id     | VARCHAR(50) | ユーザーID（FK） |
| date        | DATE        | 利用日        |
| method      | VARCHAR(50) | 交通手段       |
| departure   | TEXT        | 出発地        |
| arrival     | TEXT        | 到着地        |
| via         | TEXT        | 経由地        |
| amount      | INT         | 金額         |
| note        | TEXT        | 備考         |
| created_at  | TIMESTAMP   | 登録日時       |

---

### ■ t_keihi（経費テーブル）

経費申請情報を管理するテーブル。

| カラム名       | 型           | 説明         |
| ---------- | ----------- | ---------- |
| keihi_id   | SERIAL      | ID（PK）     |
| user_id    | VARCHAR(50) | ユーザーID（FK） |
| date       | DATE        | 利用日        |
| method     | VARCHAR(50) | 手段         |
| departure  | TEXT        | 出発地        |
| arrival    | TEXT        | 到着地        |
| via        | TEXT        | 経由地        |
| amount     | INT         | 金額         |
| note       | TEXT        | 備考         |
| created_at | TIMESTAMP   | 登録日時       |

---

### ■ t_application（申請テーブル）

各種申請（勤怠・交通費・経費）の申請状況を管理するテーブル。

| カラム名           | 型           | 説明                              |
| -------------- | ----------- | ------------------------------- |
| application_id | SERIAL      | 申請ID（PK）                        |
| user_id        | VARCHAR(50) | ユーザーID（FK）                      |
| category       | VARCHAR(20) | 申請種別（KINTAI / KOUTSUHI / KEIHI） |
| year_month     | CHAR(7)     | 対象年月（YYYY-MM）                   |
| status         | VARCHAR(20) | 申請状態（申請中・承認・差戻など）               |
| comment        | TEXT        | コメント                            |
| created_at     | TIMESTAMP   | 登録日時                            |
| updated_at     | TIMESTAMP   | 更新日時                            |

---

## ■ リレーション概要

* m_user を親として各テーブルが user_id で紐づく
* t_kintai は「ユーザー × 日付」で一意管理
* t_application は申請単位でステータス管理を行う
